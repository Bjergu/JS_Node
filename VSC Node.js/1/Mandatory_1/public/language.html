Types

- var declarations are globally scoped or function scoped while let and const are block scoped.

- var variables can be updated and re-declared within its scope; let variables can be updated but not re-declared; const variables can neither be updated nor re-declared.

- They are all hoisted to the top of their scope. But while var variables are initialized with undefined, let and const variables are not initialized.

- While var and let can be declared without being initialized, const must be initialized during declaration.


You can find more about types on this page: https://www.freecodecamp.org/news/var-let-and-const-whats-the-difference/

Environment value

Environment variables are the way to configure diffrerent aspects of Node.js applications. That can be for example env.Port.

While hosting your application on another service ((environments) like Heroku, aws etc.), your host may independently configure the "process.env.PORT" variable.

Package.json

Package.json is grat thing to make it easier for others to maage and install your package. It contains a list of the packages(Dependencies) your project depends on, specifies versions of packages and makes your build reproducible, and therfore easier to share with other developers.

Dependencies and scripts are JSON objects containing nested JSON object as value.
